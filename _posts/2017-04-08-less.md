# 变量

> 将常用的值定义在一个地方。

## 概览

我们经常在代码中看到同一个值多次（也可能是上百次）重复出现在样式表的各个地方，变量的作用就是把值定义在一个地方，然后在各处使用，这样能让代码更易维护：

```less
.transition(@transition) {
  -webkit-transition: @transition;
  -moz-transition: @transition;
  -o-transition: @transition;
  transition: @transition;
}
.opacity(@opacity) {
  opacity: @opacity / 100;
  filter: ~"alpha(opacity=@{opacity})";
}

a {
 .transition(all 0.4s);
  &:hover {
  .opacity(70);
  }
}

@theGoodThings: ~".food, .beer, .sleep, .javascript";

@{theGoodThings} {
  font-weight: bold;
}
```

输出

```css
a {
  -webkit-transition: all 0.4s;
  -moz-transition: all 0.4s;
  -o-transition: all 0.4s;
  transition: all 0.4s;
}
a:hover {
  opacity: 0.7;
  filter: alpha(opacity=70);
}
.food, .beer, .sleep, .javascript {
  font-weight: bold;
}

```

## 变量的使用场景

### 选择器

```less
@mySelector: banner;

.@{mySelector} {font-weight: bold;}
```

### URL

```less
@images: "../img";

body {background: url("@{images}/white-sand.png");}
```

### 导入

```less
@themes: "../../src/themes";

@import "@{themes}/tidal-wave.less";
```

### 属性

```less
@property: color;

.widget {@{property}: #0ee; background-@{property}: #999;}
```

### 变量名

变量的值可以作为其他变量的变量名，如：

```less
@fnord:  "I am fnord.";
@var:    "fnord";

.clearfix {
  &:after {
    content: @@var;
  }
}
```

其中 `@@var` 等价于` @(@var)` 等价于 `@(fnord)` 等价于 `@fnord`，所以输出：

```css
.clearfix:after {
  content: "I am fnord.";
}
```

### 延迟加载

变量也可以在定义前使用，如：

```less
.lazy-eval {
  width: @var; // @var 在下面定义
}

@var: @a;
@a: 9%;
```

当一个变量定义多次时，会从当前范围`向上搜索`，使用最后一次定义的值，类似 css 中对同一个元素定义不同的样式，最后定义的生效，如：

```less
@var: 0; // scope1, scope2
.class1 {
  @var: 1;
  .class {
    @var: 2;
    three: @var; // 当前范围内的 @var 值为 3
    @var: 3; // scope1_end
  }
  @var: 4;
  five: @var; // 当前范围内的 @var 值为 5
  @var: 5; // scope2_end
}
```

输出：

```css
.class1 {
  five: 5;
}
.class1 .class {
  three: 3;
}
```

使用导入文件时重新定义的变量会覆盖导入文件中的同名变量，导入文件内将使用新定义的变量值，@dark-color 的最终值为 dark red
 
```less
// library.less
@base-color: green; // 默认值
@dark-color: darken(@base-color, 10%);

// use of library
@import "library.less";
@base-color: red; // 新值
```

# 继承

继承是 less 的伪类，继承的选择器会附加到基选择器的规则集下，无论基选择器在什么地方，如：

```less
.inline {
  color: red;
}
nav ul {
  &:extend(.inline);
  color: blue;
}
```

选择器 nav ul 继承了选择器 .inline，输出：

```css
.inline,
nav ul {
  color: red;
}
nav ul {
  color: blue;
}
```

若 .inline 定义在 nav ul 的上面，则输出：

```css
nav ul {
  color: blue;
}
.inline,
nav ul {
  color: red;
}
```

即：继承的选择器 nav ul 总会被附加到基选择器 .inline 所在的位置。

所以当选择器有自己的样式时需将基选择器放到前面，否则会覆盖自己的样式。

## 继承语法

简单继承，如下：

```less
.a:extend(.b) {}
// 同下面写法
.a {
  &:extend(.b);
}
```

继承和基选择器相关的所有样式，如下：

```less
.c:extend(.d all) {
  // extends all instances of ".d" e.g. ".x.d" or ".d.x"
}
// 同下面写法
.c:extend(.d) {
  // extends only instances where the selector will be output as just ".d"
}
```

继承多个基类，中间用逗号分割，如下：

```less
.e:extend(.f) {}
.e:extend(.g) {}
// 同下面写法
.e:extend(.f, .g) {}
```

## 继承添加到选择器的方式

* 继承紧接着选择器：pre:hover:extend(div pre)
* 继承和选择器之前可以有空格：pre:hover :extend(div pre)
* 继承可以串联多个：pre:hover:extend(div pre):extend(.bucket tr)
同：pre:hover:extend(div pre, .bucket tr)
* 继承必须在选择器的最后，之后不能再有选择器，如下面写法是错误的：
pre:hover:extend(div pre).nth-child(odd)
即：继承后面不能再写选择器 .nth-child(odd)

规则集中的选择器使用继承，如下：

```less
.bag{
  color: red;
}
.bucket{
  color: green;
}

.big-division,
.big-bag:extend(.bag),
.big-bucket:extend(.bucket) {
  color: blue;
}
```

输出：

```less
.bag,
.big-bag {
  color: red;
}
.bucket,
.big-bucket {
  color: green;
}
.big-division,
.big-bag,
.big-bucket {
  color: blue;
}
```

### 在规则集样式内使用继承  Extend inside ruleset

继承可以使用 `&:extend(selector)` 语法放在规则集样式内，他是应用到多个选择器的快捷方式，如下：

```less
pre:hover,
.some-class {
  &:extend(div pre);
}
```

等价于：

```less
pre:hover:extend(div pre),
.some-class:extend(div pre) {}
```
